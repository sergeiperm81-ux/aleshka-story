# Возвращение Волшебной книги

MVP-прототип интерактивной сказки на `React + Vite`.

## Что уже есть

- библиотека с карточкой истории и заделом под будущие главы
- линейное чтение первой главы по экранам
- катсцена в формате слайд-шоу
- мини-игра на сбор волшебных книг
- настоящая развилка с выбором еды для Волшебника
- сохранение прогресса в `localStorage`
- простые звуки интерфейса без отдельных аудиофайлов
- заглушки под будущие иллюстрации

## Локальный запуск

```powershell
npm.cmd install
npm.cmd run dev
```

## Сборка

```powershell
npm.cmd run build
```

Готовые статические файлы появятся в папке `dist`.

## Деплой на VPS

Подход для надежного MVP без Cloudflare:

1. Собрать проект локально командой `npm.cmd run build`.
2. Перенести содержимое папки `dist` на VPS.
3. Раздавать папку `dist` через `nginx` как обычный статический сайт.
4. Позже подключить домен и HTTPS.

Готовые файлы для Hetzner:

- `deploy/nginx/aleshka-story.conf`
- `deploy/README-HETZNER.md`
- `deploy/package-dist.ps1`

## GitHub Pages

Для бесплатного временного деплоя проект уже подготовлен:

- добавлен `vite.config.js` с относительным `base`
- добавлен workflow `.github/workflows/deploy-pages.yml`

Что нужно сделать:

1. Создать новый репозиторий на GitHub.
2. Загрузить в него этот проект.
3. Запушить код в ветку `main`.
4. На GitHub открыть `Settings -> Pages`.
5. В `Source` выбрать `GitHub Actions`.
6. После первого пуша дождаться завершения workflow `Deploy To GitHub Pages`.

После этого GitHub выдаст адрес сайта вида:

```text
https://YOUR-NAME.github.io/YOUR-REPO/
```
